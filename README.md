# ♞ Self-Replicating Chess

Шахматы поверх Beagle: приложение хранит код и данные в системе контроля версий, можно реплицировать и синхронизировать.

## Как запустить

Нужен HTTP-сервер (приложение ходит по `fetch` к `/games/...`).

### Вариант 1: Node-сервер (рекомендуется — сохранение на диск и игра по сети)

```bash
cd /путь/к/chess
npm install
npm start
```

Сервер слушает порт **3000** и раздаёт приложение + сохраняет ходы в папку `games/`.  
- На этом компьютере: **http://localhost:3000/**  
- **С другого устройства** (телефон, планшет, другой ПК в той же Wi‑Fi): открой в браузере **http://\<IP-компьютера\>:3000/** — IP узнай командой `ifconfig` (macOS/Linux) или `ipconfig` (Windows), смотри адрес вида `192.168.x.x`. Подробнее в разделе [Игра по сети](#игра-по-сети-два-устройства) ниже.

### Вариант 2: Только статика (без сохранения ходов на сервер)

```bash
python3 -m http.server 8000
# или
npx serve . -p 8000
```

Открыть: **http://localhost:8000/**  
Конкретная партия: **http://localhost:8000/?game=test_game_001**

**Через Beagle:** разложить файлы в Beagle и открывать `index.html` через HTTP API.

## Игра по сети (два устройства)

Чтобы играть с телефона/планшета или с другого компьютера в одной сети:

1. **Запусти Node-сервер** на том компьютере, где лежит проект:
   ```bash
   npm install
   npm start
   ```

2. **Узнай IP этого компьютера** в локальной сети:
   - **macOS / Linux:** в терминале `ifconfig` или `ip addr` — смотри адрес типа `192.168.x.x` или `10.0.x.x`.
   - **Windows:** `ipconfig` — смотри «IPv4-адрес».

3. **На втором устройстве** открой в браузере:
   ```
   http://<IP>:3000/
   ```
   Например: **http://192.168.1.100:3000/**

4. **Одна и та же партия на обоих устройствах:** открой одну и ту же игру по одному адресу:
   - На первом: **http://192.168.1.100:3000/?game=template**
   - На втором: **http://192.168.1.100:3000/?game=template**  
   Или создайте новую игру (New Game) и скопируйте ссылку из адресной строки на второй экран — тогда оба будут в одной партии. После каждого хода на одном устройстве на втором нажми **Push/Pull** (или обнови страницу), чтобы подтянуть новые ходы.

**Если с другого устройства пишет «Page not found» или страница не открывается:**

1. **Фаервол на Mac.** Разреши входящие подключения на порт 3000:
   - **Системные настройки** → **Сеть** → **Фаервол** (или **Конфиденциальность и безопасность** → **Фаервол**).
   - Включи опцию «Разрешить входящие для node» или добавь правило для порта 3000. Либо временно отключи фаервол для проверки.

2. **Одна сеть.** Оба устройства должны быть в одной Wi‑Fi (или один по кабелю в ту же сеть). Телефон через мобильный интернет до твоего домашнего IP не достучится.

3. **Проверь с самого Mac:** в браузере открой `http://localhost:3000/` — если тут всё открывается, а по `http://10.214.135.90:3000/` с телефона нет, скорее всего фаервол или роутер блокирует доступ.

4. При запуске `npm start` в терминале теперь выводится строка **Network: http://…** — это и есть адрес для другого устройства.

Порт по умолчанию — 3000. Чтобы сменить: `PORT=8080 npm start`.

## Сохранение при перезагрузке страницы

- **В браузере:** состояние партии (доска, ходы) сохраняется в **localStorage**. После перезагрузки страницы та же игра (тот же `?game=...`) откроется с последней позицией.
- **На сервере:** при запуске через **Node-сервер** (`npm start`) ходы дополнительно пишутся в папку `games/<id>/moves/`. Тогда при открытии с другого устройства или после очистки localStorage партия подгружается с сервера.

Игра задаётся параметром `?game=<id>` (по умолчанию `template`). Каждый ход — GET/PUT к `/games/<id>/moves/NNNN.json`.

## Структура

- `index.html` — страница, параметр `?game=ID`
- `chess.js` — логика шахмат, загрузка/сохранение через fetch и localStorage
- `server.js` — Node-сервер для раздачи статики и приёма GET/PUT ходов
- `replicator.js` — клонирование приложения (quine)
- `manifest.json` — описание приложения для Beagle
- `games/<id>/moves/NNNN.json` — ходы партии (создаются при игре через Node-сервер)
